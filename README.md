# 2024 MathorCup D题项目 README

## 项目简介

本项目用于整理并求解 `2024 MathorCup 数学应用挑战赛 D 题：量子计算在矿山设备配置及运营中的建模应用`。

整合包的目标有两件事：

- 把 D 题相关题面、附件、旧代码和最终代码统一收纳到一个目录下。
- 提供一套可以本地直接运行、能稳定复现问题 1 到问题 3 结果的求解脚本。

当前版本采用的是与题意等价的整数线性规划建模方式，重点是保证结果清晰、可验证、可复现。虽然题目背景强调 QUBO / Kaiwu SDK，但在本地复现实验时，整数规划更稳，也更方便核对答案。

## 建模思路

### 问题1

问题 1 是一个采购优化问题。

- 决策变量：`x_i` 表示第 `i` 类挖掘机购买数量。
- 目标函数：最大化长期利润折现值。
- 约束条件：
  - 总采购金额不超过 `2400 万元`
  - 至少选购 `3` 种挖掘机型号

这部分本质上是一个带类型选择约束的整数规划问题。

### 问题2 与 问题3

问题 2 和问题 3 都是“采购 + 配车 + 运营收益”联合优化问题，采用统一模型处理。

核心变量包括：

- `x_i`：购买的挖掘机数量
- `u_i`：是否启用第 `i` 类挖掘机
- `y_ij`：第 `i` 类挖掘机是否固定匹配第 `j` 类矿车
- `n_ij`：分配给该挖掘机类型的矿车数量
- `e_ij`：有效作业当量

其中有效作业当量通过以下线性关系表达：

- `e_ij <= x_i`
- `r_ij * e_ij <= n_ij`

这里 `r_ij` 是题目给定的匹配系数。这样可以准确表达题目里的规则：当矿车数量不足时，挖掘机不能满负荷工作，而是按比例降效。

利润按 5 年口径计算，包含：

- 收入：`产量 × 20 元/立方米`
- 挖掘机成本：采购成本 + 5 年人工维护成本 + 燃油成本
- 矿车成本：5 年人工维护成本 + 燃油成本

## 结果概览

当前脚本运行得到的最优结果如下：

### 问题1

- 最优采购：`挖1 = 1, 挖2 = 2, 挖3 = 10, 挖4 = 0`
- 总采购成本：`2380 万元`
- 最大长期利润折现值：`58000 万元`

### 问题2

- 最优采购：`挖1 = 7, 挖2 = 7, 挖3 = 3`

  - `挖1 -> 矿1`
  - `挖2 -> 矿2`
  - `挖3 -> 矿3`
- 五年总利润：`48565.32 万元`

### 问题3

- 最优采购：`挖1 = 3, 挖2 = 3, 挖3 = 3, 挖4 = 2, 挖5 = 2, 挖6 = 1`

- 五年总利润：`22080.20 万元`

## QUBO 说明

题目原始要求强调 QUBO 建模与 Kaiwu SDK 求解。当前项目没有直接调用 Kaiwu SDK，而是先用整数规划方式完成稳定求解，原因如下：

- 本地运行门槛更低，复现更容易。
- 对当前这几个规模的数据，整数规划可以直接给出清晰可核验的最优解。
- 更适合作为论文建模、结果比对和后续 QUBO 转写的基线版本。

如果后续需要扩展成 QUBO 版本，可以按下面思路继续做：

- 将整数变量按二进制展开。
- 将预算约束、车型数量约束、矿车库存约束、唯一匹配约束改写为平方惩罚项。
- 形成标准 QUBO 目标后，再接入 Kaiwu SDK 或 subQUBO 方法求解。

## 已知假设与限制

当前实现基于以下假设：

- 作业周期统一按 `5 年`、每月 `20 天`、每天 `8 小时`。
- 采购费用只在开始时一次性计入。
- 矿车型号库存以题面给定上限为准。
- 同一类挖掘机只固定匹配一种矿车型号。
- 未直接实现 Kaiwu/QUBO 求解链路，而是使用等价优化模型求解。
